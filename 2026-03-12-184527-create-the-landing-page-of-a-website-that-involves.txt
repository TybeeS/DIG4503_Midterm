❯ I launched my project in Netlify through the TybeeS_Midterm folder being in a 
 GiHub repository. When I try to create a poll, it says Failed to execute       
'json' on 'Response': Unexpected end of JSON input                              

● The issue is architectural. Netlify is a static hosting platform — it serves
  HTML/CSS/JS files but has no way to run your Express server. When the browser
  calls fetch('/api/polls'), there's no Node process to respond, so Netlify
  returns an empty 404 page and res.json() crashes trying to parse it.

  This project needs a platform that actually runs Node.js. The easiest drop-in
  replacement that also deploys from GitHub is Render. Here's how to set it up:

  1. Add a Render config file to the project:

● Write(render.yaml)
  ⎿  Wrote 9 lines to render.yaml
     services:
       - type: web
         name: pollview
         runtime: node
         buildCommand: npm install
         startCommand: node server.js
         envVars:
           - key: PORT
             value: 3000


● 2. Deploy on Render:

  1. Go to https://render.com and sign up / log in with your GitHub account
  2. Click New → Web Service
  3. Connect your GitHub repo (TybeeS_Midterm)
  4. Render will detect render.yaml automatically — just click Deploy

  That's it. Render gives you a public URL like https://pollview.onrender.com
  where the full Express server runs.

  One thing to know about Render's free tier: the server spins down after 15
  minutes of inactivity and takes ~30 seconds to wake back up on the next
  request. The data/polls.json file also resets on each deploy or restart since
  the free tier doesn't have persistent disk. For a class project that's fine —
  if you need polls to survive restarts later, that's when we'd wire up a real
  database.

  You can also remove the Netlify deployment since it can't run this project —
  just delete the site from your Netlify dashboard to avoid confusion.
